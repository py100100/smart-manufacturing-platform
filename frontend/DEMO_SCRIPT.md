# 客户演示脚本

## 演示前准备

```bash
.\start-dev.bat

# 浏览器打开启动窗口中打印的 Frontend 地址
# 默认 http://127.0.0.1:3000；如端口被占用，脚本会自动切换到 3001-3005
```

## 登录方式

本地演示认证 — 登录会校验账号密码。

可使用默认管理员账号，或在注册页创建新账号后登录。

## 一键生成演示数据

登录后进入运营总览驾驶台，点击 **"一键生成演示数据"** 按钮。

生成内容：
- 5 条单智能体场景（生产调度/质量/设备/供应链/工艺）
- 1 条三智能体全链路协同
- 涵盖预警、工单、报告、行动项

数据存储在浏览器 localStorage，不覆盖用户已有数据。

## 演示路线（推荐 15 分钟）

### 第一站：登录 → 运营总览 (2 分钟)

1. 打开启动窗口中打印的 `Frontend` 地址
2. 输入用户名密码登录
3. 自动进入运营总览驾驶台

**讲解要点：**
- 这是智能制造平台的统一入口
- 顶部的 7 个核心指标反映工厂实时状态
- 风险概览展示严重/警告/信息三类预警
- 智能体运行分布显示五个智能体调用频次
- 下方展示最近的工单、报告、行动项和协同链路

### 第二站：生成演示数据 (1 分钟)

1. 点击顶部的 **"🚀 一键生成演示数据"** 按钮
2. 指标数字立即变化
3. 风险概览出现 3 条预警（含 1 条严重）
4. 智能体分布条更新

**讲解要点：**
- 演示数据模拟了真实工厂的 5 类典型业务场景
- 所有数据与真实智能体调用结果结构完全一致
- 可随时点击"清空历史"重置

### 第三站：业务闭环中心 (3 分钟)

1. 点击左侧导航 **"业务闭环中心"**
2. 切换到"预警"Tab — 展示 8+ 条预警
3. 按等级筛选：选择"严重"
4. 切换到"工单"Tab — 展示 5 张工单
5. 按类型筛选：选择"维护"
6. 切换到"报告"Tab — 展开查看发现项和建议项
7. 切换到"行动项"Tab — 按优先级或状态筛选

**讲解要点：**
- 所有智能体产出自动汇集到闭环中心
- 支持按等级/类型/状态/智能体筛选
- 支持关键字搜索
- 不需要人工汇总，系统自动聚合

### 第四站：智能体工作台 (3 分钟)

1. 点击左侧导航 **"智能体工作台"**
2. 选择智能体下拉留空（自动路由模式）
3. 点击一个演示场景按钮，输入框自动填充
4. 点击 **"执行分析"** 或 Ctrl+Enter
5. 查看结构化结果

**讲解要点：**
- 支持自动路由和指定智能体两种模式
- 自动路由根据问题内容智能匹配最合适的智能体
- 多智能体协同：输入跨域问题自动触发链式执行
- 结果展示：概览卡片 → 摘要 → 证据 → 执行链路 → 节点反馈 → 业务闭环

### 第五站：协同分析演示 (2 分钟)

1. 输入（或点击演示场景）：
   > "质量缺陷率上升，设备振动传感器报警，需要优化工艺参数改善良品率"
2. 点击执行
3. 观察 `execution_mode = collaborative`，3 步链式执行
4. 展开每个步骤查看详情
5. 查看聚合的业务闭环

**讲解要点：**
- 一次输入自动触发质量→设备→工艺三步协同
- 每个步骤有独立摘要、证据、节点反馈
- 最终产出综合报告和聚合行动项

### 第六站：业务页面 (2 分钟)

1. 点击 **"生产调度看板"** → 指标 + 执行分析
2. 点击 **"质量检测中心"** → 不良率 + 缺陷分析
3. 点击 **"设备监控面板"** → 在线设备 + 维护工单

**讲解要点：**
- 每个业务页面预配置了对应的智能体
- 指标区根据最新分析结果动态更新
- 点击执行即调用对应智能体

### 第七站：历史记录 (1 分钟)

1. 点击左侧 **"历史记录"**
2. 查看演示数据和手动执行的记录
3. 点击任意记录展开详情
4. 强调：刷新页面数据不丢失

## 常见问题

**Q: 演示数据会覆盖我的真实数据吗？**
A: 不会。演示数据追加到历史记录，可随时清空。

**Q: 需要网络吗？**
A: 需要本地运行后端（localhost:8000）。前端演示数据本身不需要网络。

**Q: 如何重置？**
A: 在运营总览页面点击"清空历史"按钮，或清除浏览器 localStorage。

**Q: 支持哪些浏览器？**
A: Chrome、Edge、Firefox 最新版。
